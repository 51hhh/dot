# rewrite: 中文化历史提交信息

## 目标

重写当前 `master` 分支提交信息，让历史记录使用 `feat`、`fix`、`docs`、`test`、`refactor`、`chore` 等约定类型，并在冒号后使用中文说明。

## 已知信息

* 用户要求历史提交信息使用 `feat`、`fix` 等前缀，并添加中文说明。
* 用户明确要求提交说明不要使用英文。
* 当前 `master` 原有 40 个提交。
* 部分旧提交已经是中文说明，但近期多条提交仍包含英文说明。
* 重写历史会改变提交哈希；如果分支已共享，需要协调强制推送。

## 要求

* 重写当前 `master` 历史提交信息。
* 使用约定提交类型。
* 类型词保留为 `feat`、`fix`、`docs`、`test`、`refactor`、`chore`。
* 冒号后的说明只使用中文。
* 不修改源码树内容。
* 保留提交顺序、作者、日期和文件改动。

## 验收标准

* [x] 当前 `master` 所有提交都在约定提交前缀后使用中文说明。
* [x] 重写后的提交主题在冒号后不含英文字符。
* [x] 历史重写后工作树内容不变。
* [x] 已展示新的提交日志供验证。

## 风险

* 此操作会重写历史并改变提交哈希。
* 如存在远程分支，需要协调强制推送。
* 基于旧哈希的工作树或分支可能需要重新变基。
