- **Query**: SSH one-click configuration and hardening features for dot CLI tool
- **Scope**: Mixed (internal codebase analysis + external best practices)

## Overview

This research defines the features and structure for an SSH one-click configuration module in the "dot" CLI tool, which generates self-contained interactive Bash installer scripts from YAML configs.

```bash

# Write new config
```

## 2. SSH Feature Categories

### 2.1 SSH Server (sshd) Setup & Hardening

#### Core Security Settings

| Setting | Recommended Value | Purpose |
|---------|-------------------|---------|
| `PasswordAuthentication` | `no` | Disable password-based login |
| `PubkeyAuthentication` | `yes` | Enable public key authentication |
| `PermitRootLogin` | `no` or `prohibit-password` | Block direct root login |
| `MaxAuthTries` | `3` | Limit authentication attempts |
| `LoginGraceTime` | `30` | Timeout for authentication (seconds) |
| `PermitEmptyPasswords` | `no` | Disallow empty passwords |
| `X11Forwarding` | `no` | Disable X11 forwarding (security risk) |
| `AllowTcpForwarding` | `no` | Disable TCP forwarding (unless needed) |
| `ClientAliveInterval` | `300` | Keepalive interval (seconds) |
| `ClientAliveCountMax` | `2` | Max missed keepalives before disconnect |
| `MaxSessions` | `10` | Limit concurrent sessions |
| `MaxStartups` | `10:30:60` | Limit unauthenticated connections |
| `LogLevel` | `VERBOSE` | Log key fingerprints for audit |
| `StrictModes` | `yes` | Check file permissions |
| `IgnoreRhosts` | `yes` | Ignore .rhosts files |
| `HostbasedAuthentication` | `no` | Disable host-based auth |
| `PermitUserEnvironment` | `no` | Block user environment manipulation |

#### Cipher Suites (Modern - OpenSSH 6.7+)

**Source**: Mozilla SSH Guidelines, dev-sec SSH Baseline

```
# HostKey algorithms (order of preference)
HostKey /etc/ssh/ssh_host_ed25519_key
HostKey /etc/ssh/ssh_host_rsa_key
HostKey /etc/ssh/ssh_host_ecdsa_key

# Key exchange algorithms
KexAlgorithms curve25519-sha256,curve25519-sha256@libssh.org,ecdh-sha2-nistp521,ecdh-sha2-nistp384,ecdh-sha2-nistp256,diffie-hellman-group-exchange-sha256

# Ciphers (authenticated encryption preferred)
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com,aes128-gcm@openssh.com,aes256-ctr,aes192-ctr,aes128-ctr

# Message Authentication Codes (encrypt-then-MAC preferred)
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com,umac-128-etm@openssh.com,hmac-sha2-512,hmac-sha2-256,umac-128@openssh.com
```

#### Intermediate Cipher Suites (OpenSSH 5.3)

For older systems (RHEL6, CentOS6):

```
KexAlgorithms diffie-hellman-group-exchange-sha256
MACs hmac-sha2-512,hmac-sha2-256
Ciphers aes256-ctr,aes192-ctr,aes128-ctr
```

#### Diffie-Hellman Moduli

All DH moduli should be at least 3072-bit (Modern) or 2048-bit (Intermediate):

```bash
# Remove weak moduli
awk '$5 >= 3071' /etc/ssh/moduli > /etc/ssh/moduli.tmp && mv /etc/ssh/moduli.tmp /etc/ssh/moduli
```

### 2.2 SSH Client Key Generation

#### Key Types

| Type | Command | Recommendation |
|------|---------|----------------|
| **Ed25519** | `ssh-keygen -t ed25519 -C "comment"` | **Recommended** - Modern, fast, secure |
| **RSA 4096** | `ssh-keygen -t rsa -b 4096 -C "comment"` | Best compatibility |
| **ECDSA** | `ssh-keygen -t ecdsa -b 521 -C "comment"` | Elliptic curve, avoid if possible |

**Best practices**:
- Use Ed25519 when backward compatibility not required
- Use RSA 4096 when backward compatibility required
- Never use DSA (deprecated) or small RSA keys (< 2048)
- Add date to key filename: `~/.ssh/id_ed25519_$(date +%Y-%m-%d)`
- Use descriptive comments: `-C "user@host-purpose"`
- Protect with strong passphrase
- Create separate keys for different purposes

### 2.3 SSH Agent Configuration

```bash
# Start ssh-agent
eval "$(ssh-agent -s)"

# Add keys
ssh-add ~/.ssh/id_ed25519

# List loaded keys
ssh-add -l

# Configure auto-add in ~/.ssh/config
Host *
    AddKeysToAgent yes
    UseKeychain yes  # macOS only
```

### 2.4 SSH Config File (~/.ssh/config) Management

#### Global Settings

```
Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
    AddKeysToAgent yes
    HashKnownHosts yes
    IdentityFile ~/.ssh/id_ed25519
```

#### Per-Host Configuration

```
Host myserver
    HostName 192.168.1.100
    User myuser
    Port 2222
    IdentityFile ~/.ssh/myserver_key
    ForwardAgent no
```

#### Security Settings

```
# Ensure KnownHosts are unreadable if leaked
HashKnownHosts yes

# Host keys the client accepts (order honored)
HostKeyAlgorithms ssh-ed25519-cert-v01@openssh.com,ssh-rsa-cert-v01@openssh.com,ssh-ed25519,ssh-rsa
```

### 2.6 Firewall Considerations (UFW)

```bash
# Allow SSH on custom port
ufw allow 2222/tcp comment 'SSH'

# Rate limiting
ufw limit 2222/tcp comment 'SSH rate limit'

# Enable firewall
ufw enable
```

**Note**: On cloud instances (AWS, GCP), security groups handle firewall - UFW may conflict.

## 3. Edge Cases & Special Considerations

### 3.1 Cloud Instances (AWS, GCP, Azure)

- Default to key-based auth (password auth already disabled)
- May need `PermitRootLogin prohibit-password` for initial setup
- Security groups handle firewall (don't enable UFW)
- Preserve default key pair provided by cloud provider
- User-data scripts may need SSH access during provisioning

### 3.2 Headless Servers

- No desktop environment
- Consider screen/tmux for session management
- May need unattended upgrades for security patches
- Serial console access as fallback

### 3.3 Desktop Machines

- SSH agent integration with desktop environment
- GUI SSH clients available
- Keychain integration (GNOME Keyring, macOS Keychain)

### 3.4 Existing SSH Config Conflicts

**Strategy**: Always backup before modification

```bash
# Backup with timestamp
cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.$(date +%Y%m%d%H%M%S)

# Validate before restart
sshd -t

# Restart safely
systemctl restart sshd
```

**Considerations**:
- Preserve custom settings
- Merge rather than overwrite when possible
- Test connection before closing current session
- Keep backup of authorized_keys

### 3.5 Non-Standard Ports

- Change from 22 to custom port (e.g., 2222)
- Update firewall rules
- Document the port prominently
- Consider port knocking for additional security
- Update client config

### 3.6 Multi-Factor Authentication (MFA)

OpenSSH 6.3+ supports MFA:
- OATH Toolkit (TOTP/HOTP)
- DuoSecurity
- Google Authenticator (libpam-google-authenticator)

**Warning**: Keyboard-interactive must be enabled, which may allow password auth - configure PAM carefully.

## 6. Template Script Implementation Guidelines

### 6.1 Safety Patterns

**Always backup before modifying**:
```bash
backup_file() {
  local file="$1"
  if [ -f "$file" ]; then
    cp "$file" "${file}.bak.$(date +%Y%m%d%H%M%S)"
    log_info "已备份 $file"
  fi
}
```

**Validate configuration before applying**:
```bash
# Test sshd config
if ! sshd -t; then
  log_error "sshd 配置验证失败，已恢复备份"
  cp "${SSHD_CONFIG}.bak.*" "$SSHD_CONFIG"
  return 1
fi
```

**Test connection before closing session**:
```bash
log_warn "请在新终端测试 SSH 连接后再关闭此会话"
log_warn "测试命令: ssh -p $NEW_PORT user@host"
```

### 6.2 Configuration Management

**Append to config file**:
```bash
cat >> /etc/ssh/sshd_config << 'SSHD_CONFIG'
# Security hardening - added by dot
PasswordAuthentication no
PubkeyAuthentication yes
PermitRootLogin no
MaxAuthTries 3
SSHD_CONFIG
```

**Replace specific settings**:
```bash
sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
```

### 6.3 Error Handling

```bash
if ! dot_sudo apt-get install -y openssh-server; then
  log_error "openssh-server 安装失败"
  return 1
fi

if ! dot_sudo systemctl enable ssh; then
  log_error "无法启用 SSH 服务"
  return 1
fi
```

## 7. Implementation Priorities

### Phase 1: Core Server Hardening
1. Install openssh-server
2. Disable password authentication

## 8. Caveats & Warnings

1. **Lockout Risk**: Disabling password auth without configured keys will lock out remote users. Always ensure at least one key is authorized before disabling passwords.

2. **Cloud Instances**: Security groups may block custom ports. Coordinate with cloud provider settings.

3. **Existing Sessions**: Always test new SSH connection before closing current session to avoid lockout.

4. **Systemd Socket Activation**: Debian 12+ and Ubuntu 22.04+ use systemd socket activation for SSH. The script should handle this.

5. **PAM Configuration**: MFA setups require careful PAM configuration to avoid unintended password auth.

6. **Host Key Regeneration**: If regenerating host keys, clients will see warnings. Document this.

7. **Backup Retention**: Consider cleaning old backups to avoid disk space issues.
